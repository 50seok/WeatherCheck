# 🔧 트러블슈팅 — 디스코드 봇 도움말 중복 발송

**날짜**: 2026-07-05

**상황**: 디스코드 채팅봇을 백그라운드로 재실행했더니, 사용자 메시지 하나에 도움말(및 모든 응답)이 2번씩 나옴.

**원인**: 봇 프로세스가 2개 동시에 떠서 각자 Discord Gateway에 로그인 → 메시지 하나에 두 인스턴스가 각각 응답.
1. 1차 실행: `python -m src.discord_chat_bot &` (같은 쉘 안에서 `sleep`+로그 확인까지 묶어서 실행) → 스크립트가 끝나도 프로세스는 안 죽고 살아있었음(PID 1565).
2. 로그 확인용 `ps aux | grep discord_chat_bot`이 매치 안 돼(Windows/MSYS `ps aux`가 명령행 인자를 안 보여줌) 죽은 걸로 착각.
3. "안 떠 있다"고 판단해 2차로 재실행(PID 2281) → 실제로는 둘 다 살아서 중복 응답 발생.

**해결**: `ps -ef`(전체 명령행 표시)로 재확인해 `-m src.discord_chat_bot` 프로세스가 2개(1565, 2281)인 것 확인 → 오래된 쪽(1565) 종료, 최신 로그인 확인된 프로세스(2281)만 유지.

**재발 방지**:
- Windows/MSYS 환경에서 백그라운드 파이썬 프로세스 생존 확인은 `ps aux`가 아니라 **`ps -ef`**로 명령행까지 봐야 함(`ps aux`는 인자를 안 보여줘서 오탐 위험).
- 봇처럼 계속 떠 있어야 하는 프로세스는 재실행 전에 항상 `ps -ef | grep discord_chat_bot`으로 기존 인스턴스부터 확인.
