# 🌤️ 출근 비서 프로젝트 로드맵 (ML → DL → LLM)

> **목표:** 날씨 + 교통을 학습해서, 출근 전 폰으로
> "☔ 비 오니 우산, 🌡️ 일교차 크니 겉옷, 🚇 9호선 혼잡하니 일찍" 알려주는 AI 비서 만들기
>
> **전략:** 하나의 테마(날씨)로 ML→DL→LLM을 점진적으로 쌓아 올림. 앞 단계 결과물을 다음 단계 재료로 사용.

---

## 🔑 왜 이 테마인가 — "승리 공식"

```
✅ 시계열 데이터 (시간 순서) → ML 예측 → DL(LSTM) → 자연 연결
✅ 매일 쓰는 개인 알림으로 끝남 → LLM "아침 비서"로 마무리 딱
```

---

## 🏗️ 전체 아키텍처

```
[Phase 1: ML]  과거 날씨 CSV → "내일 기온/비 예측" 모델
                    ↓ 같은 데이터, 더 정교하게
[Phase 2: DL]  LSTM → "지난 7일로 내일~3일 시계열 예측" (정확도 ↑)
                    ↓ 예측 결과를 가져다 씀
[Phase 3: LLM] DL예측(날씨) + 교통API(혼잡/정체)
                → LLM이 자연어 브리핑 생성
                → 📱 출근 전 폰 알림
   "☔ 오늘 비 90%, 우산 필수! 🌡️ 일교차 12도 겉옷 챙기세요.
    🚇 9호선 혼잡, 10분 일찍 나가세요"
```

---

## 📅 Phase 1 — ML (날씨 예측 모델)

| 항목 | 내용 |
|---|---|
| **데이터** | 기상청 과거 날씨 CSV (기온·습도·기압·풍속·강수량, 몇 년치) |
| **입력(X)** | 오늘의 기온, 습도, 기압, 어제 강수량 등 |
| **정답(y)** | 내일 기온(회귀) / 내일 비 여부(분류) |
| **모델** | LinearRegression → RandomForest → XGBoost (쉬운 것부터) |
| **흐름** | 분석 7단계 + ML 3단계(전처리→학습→평가) |
| **결과물** | "내일 기온 18도, 강수확률 70%" 예측 + 정확도 |
| **배우는 것** | 회귀/분류, train_test_split, 모델 평가(정확도·MAE) |

- 🛠️ 새 라이브러리: **scikit-learn**
- 📍 데이터: 기상자료개방포털 (data.kma.go.kr) 에서 지역·기간 골라 CSV 다운

**📊 발표 + 🚀 배포 (Streamlit)**
| | 내용 |
|---|---|
| 📊 발표자료 | EDA 시각화(heatmap·분포) + 모델별 정확도 비교 차트 + 주피터 노트북 |
| 🚀 배포 | **Streamlit 데모** — 슬라이더로 기온·습도 입력 → 내일 날씨 예측 출력 |
| 산출물 | 코드 + 발표 5장(문제→데이터→방법→결과→배운점) + Streamlit 앱 |
| 완료 기준 | Streamlit 데모가 실제로 예측값을 보여줌 |

---

## 📅 Phase 2 — DL (시계열 딥러닝)

| 항목 | 내용 |
|---|---|
| **왜 DL?** | 날씨는 "순서"가 중요(어제·그제 흐름이 내일에 영향). ML은 순서를 잘 못 봄, **LSTM은 순서를 기억** |
| **입력** | 지난 7일치 날씨 시퀀스 |
| **출력** | 내일(또는 향후 3일) 기온·강수 |
| **모델** | LSTM / GRU (순환신경망) |
| **포인트** | ML 모델과 정확도 비교 → "DL이 시계열에서 더 낫네" 체감 |
| **결과물** | 며칠치 예보를 한 번에, 더 정확하게 |
| **배우는 것** | 시계열 데이터 만들기, 신경망 구조, 학습 곡선 |

- 🛠️ 새 라이브러리: **TensorFlow/Keras** 또는 **PyTorch**
- 💡 여기서 **CSV → 실시간 API 전환**: 기상청 API로 최신 데이터 받아 예측

**📊 발표 + 🚀 배포 (Streamlit)**
| | 내용 |
|---|---|
| 📊 발표자료 | 학습곡선 + **"ML vs DL 정확도 비교"** + 예측 그래프 |
| 🚀 배포 | **Streamlit 데모** — Phase 1 앱 확장, 며칠치 예보 그래프 + 실시간 API 연동 |
| 산출물 | 코드 + 모델 저장(.h5) + 발표 5장 + 업그레이드된 Streamlit 앱 |
| 완료 기준 | Streamlit에서 ML/DL 예측을 나란히 비교해서 보여줌 |

---

## 📅 Phase 3 — LLM (통합 출근 비서) ⭐

> **핵심:** LLM은 날씨를 "예측"하지 않는다. Phase 1·2가 예측한 걸 받아 **사람 말로 조언**하고 **알림**을 보낸다.

### 들어가는 재료 3가지
| 재료 | 출처 | 예시 |
|---|---|---|
| 🌤️ 날씨 예측 | Phase 2의 DL 모델 | 강수 90%, 최고 25도/최저 13도 |
| 🚇 교통 현황 | 교통 API | 9호선 혼잡 / 외곽순환 정체 |
| ⚙️ 조언 규칙 | 내가 정한 로직 | 아래 표 |

### LLM이 만드는 조언 로직
| 조건 | 알림 문구 |
|---|---|
| 강수확률 높음 | "☔ 우산 챙기세요" |
| 일교차 큼(최고-최저 차이↑) | "🌡️ 겉옷 챙기세요" |
| 미세먼지 나쁨 | "😷 마스크 챙기세요" |
| 대중교통 혼잡 | "🚇 10분 일찍 나가세요" |
| 자차 + 도로 정체 | "🚗 우회도로 추천, 30분 소요" |

### 출퇴근 모드 분기
```
대중교통 → 지하철/버스 혼잡도·도착정보 알림
자차    → 도로 정체·예상 소요시간 알림
```

### 동작 흐름 (매일 아침 자동)
```
1. 스케줄러가 아침 7시에 실행
2. DL 모델 호출 → 오늘 날씨 예측
3. 교통 API 호출 → 출근길 현황
4. LLM에게 전달 → "이 데이터로 출근 브리핑 써줘"
5. LLM이 자연어 생성 → 폰 알림 발송 📱
```

| 항목 | 내용 |
|---|---|
| **LLM 역할** | 숫자 예측 → 자연스러운 한국어 브리핑으로 변환 |
| **알림 수단** | 디스코드 봇 / 카톡 / 폰 푸시 (쉬운 것부터, 디스코드 추천) |
| **자동화** | 스케줄러(cron)로 매일 아침 자동 실행 |
| **배우는 것** | LLM API(프롬프트·function calling), API 연동, 스케줄링, 알림 |

- 🛠️ 새 기술: **Claude/OpenAI API** + **스케줄러** + **알림 봇**

**📊 발표 + 🚀 배포 (Streamlit + 실서비스)**
| | 내용 |
|---|---|
| 📊 발표자료 | 아키텍처 다이어그램 + **실제 알림 데모 영상/스크린샷** |
| 🚀 배포 | **Streamlit 대시보드** (오늘 브리핑·예보·교통 한 화면) + **디스코드 봇 실서비스** (매일 아침 자동 알림) |
| 산출물 | 코드 + 발표 5장 + Streamlit 대시보드 + 작동하는 알림 봇 |
| 완료 기준 | 매일 아침 폰으로 자연어 출근 브리핑이 자동 도착 |

> 💡 최종 배포 옵션: Streamlit Community Cloud(무료·가장 쉬움) 또는 OCI 서버(oci-arm2, 자동배포 세팅됨 — 그때 상태 확인 필요)

## 🗺️ 단계별 요약표

| | Phase 1 (ML) | Phase 2 (DL) | Phase 3 (LLM) |
|---|---|---|---|
| **목표** | 날씨 예측 | 시계열로 더 정확히 | 말로 조언+알림 |
| **데이터** | 과거 CSV | CSV→실시간 API | DL예측+교통API |
| **핵심 기술** | scikit-learn | LSTM(Keras) | LLM API+스케줄러 |
| **결과물** | 예측 모델 | 정교한 예보 | 📱 출근 비서 알림 |
| **📊 발표** | EDA+정확도 차트 | ML vs DL 비교 | 아키텍처+데모영상 |
| **🚀 배포** | Streamlit 예측 데모 | Streamlit 예보 확장 | Streamlit 대시보드+디스코드 봇 |
| **난이도** | ⭐ | ⭐⭐⭐ | ⭐⭐⭐ |

> **배포는 전부 Streamlit 통일** (Phase 1 데모 → 점진 확장 → Phase 3 대시보드). 발표는 매 단계 5장(문제→데이터→방법→결과→배운점), 3개 모이면 전체 포트폴리오.
